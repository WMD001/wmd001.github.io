{"posts":[{"title":"Gridea 同步问题","content":"🤓GitHub同步失败问题解决 😎借助gitee仓库镜像功能 问题描述 Gridea 同步到 github pages 的时候，总会遇到同步失败的情况。 解决办法 可以通过 Gitee 仓库镜像功能来间接实现同步到github。 前提准备 Gitee 上建立一个仓库，用于同步代码 GitHub 上建立一个仓库，开启 pages 功能 配置镜像同步 Gitee 选择仓库，管理，仓库镜像管理（官方文档上说这个功能是限时开放的，限时到2022年8月31日，但是现在已经过期了，我也没有找到另外的说明，所以还是能用的） 选择添加镜像， 之后选择gitee -> github方向的push。 配置Gridea Gridea 配置远程，选择GiteePages，填写配置，测试连接后保存，之后将域名修改为github pages 的地址，保存。 至此，已经大功告成了。 同步数据 直接点击同步按钮就可以，修改域名是为了修改生成html代码里css的引用路径，同步成功后，登陆gitee查看镜像同步情况，更新成功说明已经推送到github。 可能的问题 gitee 更新到github失败，由于gitee推送到github是强制覆盖推送，所以对于受保护分支会失败。可以设置别的分支或者修改保护分支设置 Settings -> Branches -> Branch portection rules -> Edit 设置允许强制推送，之后点击手动更新，或者gridea上点击同步，稍等片刻，就能看到成功推送到github。 ","link":"https://wmd001.github.io/post/gridea-tong-bu-wen-ti/"}]}
